# Code of Conduct

Pillars is a small community focused on building a useful standard. We aim for the basics: be respectful, be constructive, be patient with newcomers. These principles guide all participation in this project's issues, discussions, pull requests, and any related forums.

## Expected behavior

- **Be respectful.** Treat others' time, work, and viewpoints with care. Critique ideas, not people.
- **Be constructive.** When you disagree, propose a path forward. "This is bad" is less useful than "This breaks because X; consider Y instead."
- **Be patient.** Pillars covers conceptual ground (standards, agents, conventions) that has a learning curve. Help newcomers; don't shame them for asking.
- **Be clear.** Spec discussions require precision. If you mean "in the typical case," say so. If you have a specific failure mode in mind, describe it.

## Unacceptable behavior

- Personal attacks, harassment, or discrimination of any kind.
- Sustained disruption of issues or PRs (off-topic tangents, repeated bad-faith arguments after consensus).
- Publishing others' private information without consent.
- Sexualized language or imagery, or unwelcome sexual attention.
- Other conduct that a reasonable participant would consider inappropriate in a professional setting.

## Scope

This code applies to all project spaces: issues, pull requests, discussions, project email, and any official forums.

## Enforcement

Concerns can be raised by opening a private GitHub issue or contacting the maintainers directly through GitHub. Reports are handled confidentially. Possible responses include:

1. **Private clarification.** A maintainer reaches out to discuss the concern.
2. **Public correction.** A maintainer addresses the issue in the thread where it occurred.
3. **Temporary or permanent removal from project spaces.** For repeated or serious violations.

Maintainers exercise discretion. The goal is a productive standards community, not a punitive process.
